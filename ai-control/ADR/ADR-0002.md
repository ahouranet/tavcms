# ADR-0002: Shared Contracts (i18n/slug/dates/media/seo)

## Status
Accepted

## Context
پروژه CMS شامل چندین اپ محتوا (blog/page/product/...) است که باید رفتار یکسانی در موارد مشترک داشته باشند:
- URL prefix زبان
- استاندارد اسلاگ (فارسی/انگلیسی) با یکتایی در زبان
- نمایش تاریخ بر اساس زبان و تنظیمات تقویم
- استفاده مشترک از Media و SEO

## Decision
- قراردادهای مشترک ابتدا در اپ‌های زیرساختی پیاده می‌شوند و اپ‌های محتوا فقط مصرف‌کننده هستند:
  - `apps/core`: i18n/slug/dates/base mixins + settings singleton
  - `apps/media_core`: Asset/upload (MVP)
  - `apps/seo_core`: meta fields + sitemap (MVP)
- در MVP از ساخت سیستم‌های کامل (Yoast/media processing/page builder کامل) خودداری می‌شود و به فازهای plus منتقل می‌گردد.

## Consequences
- هماهنگی بالا بین اپ‌ها
- کاهش دوباره‌کاری و جلوگیری از divergent implementations
- توسعه تدریجی (Core → Plus → Pro) برای اپ‌های بزرگ

## Notes
- Target framework version: **Django 5.2.x (LTS)** on **Python 3.12.x**.
